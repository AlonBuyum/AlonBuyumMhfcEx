# Deployment Runbook

Step-by-step from a fresh local clone to a live, public app. Estimated time: 20-30 minutes if accounts already exist.

## 0. Prerequisites

- A GitHub account
- A Render account (free tier is enough)
- A Vercel account (free tier is enough)

## 1. Push to GitHub

```bash
cd "C:\Alon\My Documents\Mine\MHFC\crypto-advisor"
# Create the repo at github.com first, then:
git remote add origin https://github.com/<your-username>/crypto-advisor.git
git push -u origin main
```

## 2. Deploy backend + database via Render Blueprint

Render reads the committed `render.yaml` and provisions both services at once.

1. Go to https://dashboard.render.com/blueprints → **New Blueprint Instance**
2. Connect the GitHub repo → Render parses `render.yaml` automatically
3. Render will create:
   - `crypto-advisor-db` (free Postgres, 90-day expiry)
   - `crypto-advisor-api` (web service running `cd backend && npm install && npm run build && npm start`)
4. Fill in the secret Render prompts for:
   - `FRONTEND_URL` — leave blank for now, fill in after step 3
5. `JWT_SECRET` is auto-generated by Render (per the `generateValue: true` in `render.yaml`)
6. `DATABASE_URL` is wired automatically from the Postgres service
7. Click **Apply** → wait ~3-5 minutes for the first build + deploy
8. Once live, note the API URL: `https://crypto-advisor-api-<hash>.onrender.com`
9. Sanity check: `curl https://crypto-advisor-api-<hash>.onrender.com/health` should return `{"status":"ok","uptime":...}`

## 3. Deploy frontend on Vercel

1. Go to https://vercel.com/new → **Import Project**
2. Pick the GitHub repo
3. **Important:** in "Configure Project":
   - **Root Directory:** `frontend`
   - Framework Preset: Vite (auto-detected)
   - Build Command: `npm run build` (default)
   - Output Directory: `dist` (default)
4. Add environment variable:
   - `VITE_API_URL` = `https://crypto-advisor-api-<hash>.onrender.com` (the URL from step 2.8)
5. Click **Deploy** → wait ~1-2 minutes
6. Once live, note the frontend URL: `https://crypto-advisor-<hash>.vercel.app`

## 4. Wire CORS back to Render

Now that Vercel has assigned a URL, go back to Render:

1. Open the `crypto-advisor-api` service → **Environment**
2. Set `FRONTEND_URL` = `https://crypto-advisor-<hash>.vercel.app`
3. Save → Render redeploys automatically (~1 minute)

If you skip this step, the frontend will hit CORS errors when calling the API.

## 5. Create a read-only Postgres user for the reviewer

Render's Postgres dashboard provides a `psql` connection string. From your terminal:

```bash
psql "postgres://crypto_advisor:...@dpg-XXX.oregon-postgres.render.com/crypto_advisor" <<SQL
CREATE USER reviewer WITH PASSWORD 'pick-a-random-password';
GRANT CONNECT ON DATABASE crypto_advisor TO reviewer;
GRANT USAGE ON SCHEMA public TO reviewer;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO reviewer;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO reviewer;
SQL
```

The reviewer's connection string is the same host/port/db but with `reviewer:<password>` instead of the owner credentials.

## 6. Smoke test the live app

1. Open the Vercel URL → login page should render
2. Click **Create one** → register a real account (the form validates: email format, 8-char password)
3. After register, you should land on the dashboard with all 3 cards populated:

4. Click **New meme** — the meme should change
5. Hit refresh — you should stay signed in (token in localStorage)
6. Sign out → redirected to login → try to visit `/` directly → redirected back to login

## Troubleshooting

### Backend logs say `Missing required env var: JWT_SECRET`
The Blueprint should auto-generate it. If it didn't, go to Environment → add `JWT_SECRET` with any 64-char hex string.

### Frontend shows "Network Error" on login
- Check `VITE_API_URL` on Vercel — should be the Render URL, no trailing slash
- Check `FRONTEND_URL` on Render — should match the Vercel URL exactly
- Check the Render API is awake (`curl /health`) — free tier sleeps after 15 min

### `/api/meme` returns the fallback in production
- Reddit might be rate-limiting. The User-Agent header should help. If it persists, the fallback set still gives a good demo.

### Postgres connection refused
- Render's free Postgres takes ~30s to come up after the Blueprint applies. If the API service started first, restart it from the Render dashboard.

### Render free tier sleeps after 15 minutes
- First request after sleep takes ~30 seconds
- For a smoother review: leave the dashboard open just before sharing the link, OR upgrade to a paid plan
- Optional: use UptimeRobot (free) to ping `/health` every 5 min, but this is technically against Render's free-tier ToS in spirit

### Postgres 90-day expiry approaching
- Render emails you a warning at 80 days
- Take a `pg_dump`, provision a fresh DB, restore via `psql`. Update `DATABASE_URL`.
